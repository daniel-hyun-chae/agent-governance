# Session Bootstrap

## Required Reads

Before performing any task, agents must read these files to build situational awareness.
All project-specific files are under the `project/` directory.

1. `project/architecture/codebase-map.md` -- what exists and where it lives.
2. `project/architecture/domain-glossary.md` -- canonical vocabulary.
3. `project/architecture/overview.md` -- system shape, tech stack, data flow.
4. `AGENTS.md` -- agent taxonomy, workflow, source-of-truth locations.
5. `project/backlog/README.md` -- backlog conventions, current priorities.
6. `project/spec/README.md` -- how specs are structured, which files exist.

## When to Re-read

- Before touching a domain area you have not yet read into this session.
- Before renaming or redefining any concept (check the glossary first).
- When resuming work from a backlog item written in a prior session.

## Rationale

Agents lose all context between sessions. These six files give enough orientation to act without redundant questions or incorrect assumptions.
